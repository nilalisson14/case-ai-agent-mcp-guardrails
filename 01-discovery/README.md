# 01 — Discovery

**Pergunta que esta pasta responde:** qual problema de negócio justifica este projeto?

## Resumo

Um analista de conformidade da ANVF (agência reguladora fictícia, mesma do Capítulo 1) recebe pareceres técnicos já emitidos e precisa confirmar, manualmente, se o prazo concedido está de acordo com a regra da classe de risco do produto, e se retificações posteriores foram aplicadas corretamente. Hoje isso é verificação repetitiva que consome tempo de analista sênior — candidata natural a automação assistida, mantendo a decisão final humana.

Detalhamento completo do cenário, stakeholders e a lacuna identificada no Capítulo 1 que originou este projeto: ver [`../02-requirements/caso_de_uso.md`](../02-requirements/caso_de_uso.md), Seções 1 e 2.

## Por que não há mais artefatos nesta pasta

Persona, jornada e mapeamento de stakeholders já existem, em forma mais concisa, dentro do caso de uso consolidado — não duplicados aqui para evitar informação divergente entre dois documentos sobre o mesmo cenário. Se este projeto crescer além do estudo de caso, esta pasta ganha os artefatos completos (Lean Canvas, Personas, Event Storming).
